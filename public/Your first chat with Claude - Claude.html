A few likely causes, in order of probability:

1. Browser cache — most common culprit
Even after redeployment, browsers cache images/CSS aggressively. Try:

  • Hard refresh: Ctrl+Shift+R
  • Or open the site in an Incognito/Private window to rule out cache entirely

2. The edit didn't actually save before committing
Double check Header.jsx right now — does the line show your new size (e.g. h-24), or did it somehow revert to h-16? Sometimes if multiple terminal/edit sessions were open, changes get overwritten.

3. Check what's live on GitHub directly
Go to your GitHub repo → src/components/Header.jsx → confirm the actual pushed code shows your new height class.

4. Vercel deployed, but from an old commit
Check the Deployments tab — does the latest "Ready" deployment's timestamp match right after your last push? If it's showing an older time, the new push might not have triggered a fresh build.
